adnroid中不得不了解的东西

发表于 2019-03-08

Application类

  1. 每次当应用程序启动的时候，就会调用这个类，这个类是用于维护全局状态的基类（简而言之，就是设置全局变量的），可以通过创建此子类并将此子类的完全限定名称制定为android：nameAndroidManifest.xml <application>标记中的属性来提供自己的实现
  2. 如果单例需要全局上下文（例如广播接收器），则在调用单例方法时包含Context.getApplicationContext()作为Context参数getInstance()
  3. 通常情况下是系统帮我们进行创建的，如果要进行自定义Application，需要我们继承一下这个类
  4. 这个时候，就可以使用myApplication.getContext()来获取Context对象了，一个问题：为什么不可以直接用getApplicationContext()。
  5. Application和Servie、Activity一样作为android的系统组件.通过Application来获取的Context和Activity来获取的Context是不一样的，一个是系统级别的上下文环境，一个是当前这个活动级别的上下文环境，一般的UI应用设置，一般调用活动上下文进行操作。

Serializable类

  1. 我们发现，在使用Intent来启动活动、发送广播或者启动服务等，可以在Intent中添加一些数据，但是利用Intent来实现数据传递的数据类型是很有限的，所以我们可以选择自定义传递的类型，需要借助两种实现方式：Serializable和Parcelable。

  2. Serializable是一个接口，表示序列化的意思，实现这个接口可以使数据转化为可以存储或者传输的状态，序列化的对象可以在网络上进行传输。

    public Class Book implements Serializable{
    private String name;
    private String title;
    get/set function()..
} 

Intent intent = new Intent(FirstActivity.this, NextActivity.class);
intent.purExtra("book_data",Book);
startActivity(intent);

Book book = (Book) getIntent().getSerializableExtra("book_data");

Parcelable方式

  1. 实现原理和Serializable有出入，因为它是将一个完整的对象进行分解，使得分解后的每一部分的都是Intent所支持的数据类型。
  2. 需要实现Parcelable接口，其中重写describeContent()和writeToParcel()，其中writeToParcel()实现将新定义的这个实现类（如Book）中的变量写入，ParcelInstance.writeString()/.writeInt()
  3. 再定义一个创建器：Parcelable.Creator CREATOR = new Parcelable.Creator(){}，需要我们重写createFromParcel(Parcel source){Book book; book.name = source.readString();..return book;…}
  4. 最后在活动启动后，在另一个活动中获取的方法是getParcelableExtra(“key”);

implementation 和 api 以及旧版本的compile

  • 在android进入3.0后，原本的依赖库的引入由complie改为了implementation或者api
  • api可以看做是complie，完全替换没有问题
  • 而implementation则是禁止依赖的传递，官方提倡如果能用implementation，因为可以加快编译速度。

关于怎么解析从网络请求回来的json对象

  1. 在json格式的数据中，{}表示json对象JSONObject，[]表示json数组JSONArray，所以只要知道想要获取的数据被包含在怎么样的json结构中，就可以获取到数据。
    • 主要利用的类有：JSONObject JSONArray
    • 主要利用的方法有：getJSONObject(name) getJSONArray(name) getString(name)

Glide的基本用法

  1. Clide是github上的一个开源库，用来实现网络图片加载的，十分方便，不需要自己实现HTTPConnection的请求，Bitmap的生成。但是其实他的源码实现十分复杂，之后会针对这个源码来进行学习。
  2. 主要用法引入依赖implementation 'com.github.bumptech.glide:glide:3.7.0'
  3. Glide.with(context).load(url).into(ImageView)
  4. 其中上下文表示图片加载到的对象，可以包含Activity、Fragment和Context类型的参数。
